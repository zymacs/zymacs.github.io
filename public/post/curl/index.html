I recently had to test an api with a popular app called PostMan but did not like that it kept reminding me how limited the freemium version was. Not to mention the unsolicited "AI assistance" features. So I set out to make some notes on a free and possibly better alternative: Curl . This is an early version of those notes.

I'll only focus on the http protocol for this article (and will consider add notes for others when my need for using curl with them arises) but I'm sure that's for most, all they ever get to use with curl.

So what is curl?

It's a very old (26 years as of when I am writing this) popular cli tool for network data transfer by Daniel Stenberg.

All its file-transfer functionality is powered by an underlying library called libcurl.

The 'c' in Curl is for 'client' since it is a client side tool and the URL is because its a tool for 'downloading and uploading data with a url'. Okay, enough trivia.

Curl comes preinstalled on most linux distros but if it ain't installed you can install it using your distro's package manager. I am running Arch so that would be:

pacman -S curl

Since its rebelling against PostMan that birthed this article, we'll focus on what PostMan's free tier offers. Trust me, itll be enough to handle most usecases you will run into for curl.
