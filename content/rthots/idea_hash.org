Anyhow. Speaking of cosine similarity. That aside. How does sha1 work without getting all technical about it?
It generates a unique hash for a file based on its content not its meaning. Names usually convey meaning or some 
part of it. Does the brain use sha-1 for noticing what is similar to something else? Nope it doen't ... but 
its pretty good at telling 'local search engine' == 'my own internet' == 'google on localhost'.. 

On learning this the next question i had to ask is if this had been implemented at all in CS. Which it has. sematic
search finds hits for a query not based on exact matches but on meaning. The idea behind semantic search is 
cosine similarity. Okay, now I am onto something.. We are getting closer to a hypothetical sha-1 for meaning.
